# Design Principles

## 1. Playable before pretty

Build the round loop first. A fun graybox beats a beautiful empty backyard.

## 2. Fake complexity where possible

Players should feel like they are building a custom bug, but the first version should use fixed bug rigs/archetypes with safe cosmetic slots.

## 3. Clear tradeoffs

Every bug base should have an obvious strength and weakness.

## 4. Cosmetics stay cosmetic

Paid/custom skins should not make players stronger.

## 5. Kid ideas are fuel, not scope creep

Capture all ideas, but only promote the few that support the current milestone.

## 6. Hazards should be readable

Players need warnings before danger. Avoid cheap instant failure.

## 7. Small map, strong identity

One memorable backyard arena is better than five unfinished areas.
